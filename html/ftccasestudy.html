UNIVERSITY OF ARIZONA BOOTCAMP | NOV 2020

Federal Trade Commission Case Study

The Problem

While taking a UX/UI Bootcamp course with the University of Arizona, I was tasked review and redesign the UI for the FTC website. After my initial research, I discovered key problems to address in the redesign.

The Solution

In this user interface redesign, I analyzed the current FTC site and worked to make the site easier to understand, less complicated and created a cohesive look and feel throughout the site. My solution to updating the site included updating the navigation, color scheme and creating consistency throughout the site.

Timeline: Nov - Dec 2020

Responsibilities: UX Research, UI Designer

My Role: Solo Project, Site Analysis, Information Architecture, Responsive Design and Prototyping

Tools: InVision, Miro, Adobe XD, Adobe Illustrator, Adobe Photoshop

FTC Site Background

The FTC site is targeted towards consumers and people looking to protect themselves from scammers.

The purpose of the FTC’s is to direct users to take action to protect themselves as consumers. The home page contains links to file complaint or reports, read the latest news or scams, and to understand the inner workings and laws of the FTC.

FTC.gov

Federal Trade Commission’s Mission

The Federal Trade Commission’s mission is: Protecting consumers and competition by preventing anticompetitive, deceptive, and unfair business practices through law enforcement, advocacy, and education without unduly burdening legitimate business activity.

Site Evaluations

I reviewed the site’s user paths, completed a heuristic evaluation, and review the current UI patterns. I found a few issues that would hinder a users experience. I worked with a partner to review and identify key issues with the site. We also completed a color evaluation. Although the site had no conflicts in color blind mode, there were some accessibility issues with two of the colors.


Based on the heuristic principles, I included 5 of the web pages showing parts of the site and areas for improvement.

User Testing

My objective was to discover any pain points or issues users run into while using the current FTC.gov website. The goals of the user test is to learn where we can improve the site to be more user friendly and appealing to the user.

Questions to Be Answered

  • Is the site functional so that people are more likely to use and return?
  • Is it easy to use and find the information they need, so that the user is able to understand the laws and make reports?
  • Is the site appealing to the user? Is there too much information being displayed?

Tasks and Scenarios

  • Learn the date the FTC was created
  • Make a report about an online shopping fraud
  • Confirm the users number is on the “Do Not Call List”
  • Learn how to protect against phishing scams
  • View Public Comments

 

Definition

User Testing - Affinity Mapping

I analyzed the insights I received from my user testing and organized into a 2x2 Prioritization Matrix Evaluation.

Usability Tests Insights

  All of the users tested utilized the footer at some point in the test

  Very few testers used the drop down main navigation. The testers would roll over each section and skim the topics to find what they were looking for.

  One user did not use the main navigation at all, only the footer and search.

  Search and the drop down “I would like to…” were utilized often.

Card Sorting

The navigation for the FTC is robust. The naming for certain categories make sense to only those who are familiar with the FTC. My goal was to make the redesign more accessible to all users and not those who are familiar with the site.

Testing

Guerrilla User Testing of Prototype

My objective for this guerrilla test is to discover the usability of the site on both desktop and mobile to ensure the experience is similar on both. Since this is a government site, it needs to be accessible to a large number of people. So I will not be looking for specific demographic to test.

Tasks:

  • Tell me the 5 Commissioners

  • Find the Free Resource “Pass It On” on page two

Insights

  • Found prototyping issues
  • Needed some directions added to a few pages
  • With the filtering in Blogs, it was hard to see the stories changed due to size

Conclusion

Key Takeaways

Testing is important. Just because I think that a design or verbiage will work, you should still test. I thought my design made sense, but after testing, I found out that it had a few issues. Also that verbiage that I used did not always make sense.

What I Learned

Due to the large scale of the website, I learned that it is extremely important to keep my XD file clean and organized. As well as making sure I have a strong visual concept before starting the redesign.
